# BodyLog

Личный трекер: вес, дозировки (инъекции), питание, активность и аналитика с графиками.

## Стек

- Next.js 14 (App Router), TypeScript
- TailwindCSS, shadcn/ui
- Recharts, React Hook Form + Zod
- Dexie (IndexedDB), date-fns
- PWA (next-pwa)

## Запуск

```bash
npm install
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000). Данные хранятся локально в браузере (offline-first). Экспорт/импорт — в Настройках.

## Функции

- **Дашборд**: текущий вес, Δ за 7/30 дней, серия ввода, подсказка следующей инъекции, график веса за 30 дней со скользящим средним.
- **Лог**: таблица по дням (вес, ккал, макросы, дозы, активность), фильтры.
- **Аналитика**: вкладки — вес, питание (калории + макросы), дозы, активность.
- **Настройки**: профиль и цели, светлая/тёмная тема, экспорт JSON/CSV, импорт JSON с превью.

Все формы поддерживают быстрый ввод (Сегодня/Вчера), валидацию и отмену после добавления (toast).

## Данные только на вашем устройстве

Показатели (вес, питание, активность и т.д.) хранятся **локально в браузере** (IndexedDB). В репозиторий попадает только код приложения; ваши данные никуда не отправляются и не попадают в Git.

## Публикация в свой репозиторий (получить ссылку)

1. Создайте **пустой** репозиторий на GitHub / GitLab / другом сервисе (без README и без .gitignore).
2. Подключите удалённый репозиторий и отправьте код:

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git
git branch -M main
git push -u origin main
```

3. Ссылка на репозиторий будет на странице созданного репо (например: `https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ`).

## Развернуть приложение в интернете (чтобы по ссылке открывалось приложение)

### Вариант 1: GitHub Pages (всё на базе GitHub, без сторонних сервисов)

Репозиторий уже настроен на деплой через **GitHub Actions**. Чтобы по ссылке открывалось само приложение:

1. Запушьте код в репозиторий (например `main`).
2. В репозитории на GitHub откройте **Settings → Pages**.
3. В блоке **Build and deployment** выберите **Source: GitHub Actions**.
4. При каждом пуше в `main` workflow соберёт приложение и задеплоит его. Ссылка на приложение будет такой:  
   **`https://ВАШ_ЛОГИН.github.io/BodyLog/`**  
   (например: [https://kefir161-spec.github.io/BodyLog/](https://kefir161-spec.github.io/BodyLog/)).

Этой ссылкой можно делиться — откроется само приложение, а не эта страница. Данные по-прежнему хранятся только в браузере у каждого пользователя.

### Вариант 2: Vercel

Зайдите на [vercel.com](https://vercel.com), войдите через GitHub, добавьте проект из репозитория и нажмите Deploy. Получите ссылку вида `https://ваш-проект.vercel.app`.

### Вариант 3: Netlify

На [netlify.com](https://netlify.com) подключите репозиторий — получите ссылку вида `https://ваш-проект.netlify.app`.
